# 🇪🇺 CPV Code Classifier

Predicts EU **Common Procurement Vocabulary** codes for a tender description, aggregated to the **4-digit division level**. Multilingual (all 24 official EU languages), deterministic, and **hallucination-free by construction** — the classifier ranks codes from the official CPV catalogue using cosine similarity over multilingual embeddings, with no LLM in the loop.

## How it works

1. The 9,454 official CPV codes are loaded from `data/cpv_full.csv`, one row per code with descriptions in 24 languages.
2. `build_embeddings.py` precomputes a sentence embedding for every `(code, language)` pair using `paraphrase-multilingual-mpnet-base-v2` (~227k vectors of 768 dims). Output is saved to `data/cpv_embeddings.npz`.
3. At query time the app embeds the tender text into the same vector space, computes cosine similarity against every catalogue vector (a single matrix multiply, <100 ms on CPU), and aggregates scores per CPV code by taking the best-matching language.
4. Results are rolled up to the 4-digit division level and any division with a score above the configurable threshold is returned.

Because the candidate set is the closed set of real CPV codes in the CSV, **the app cannot output a code that does not exist**.

## Local setup

```bash
python -m venv .venv
source .venv/bin/activate         # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# One-off — produces data/cpv_embeddings.npz (~150–250 MB compressed):
python build_embeddings.py

streamlit run app.py
```

The first run downloads the embedding model (~420 MB) into your HuggingFace cache. Subsequent runs are instant.

## Deploying to Streamlit Cloud

1. Push this repo to GitHub. The embeddings file is large (~150–250 MB compressed); use **Git LFS** for `data/cpv_embeddings.npz`:
   ```bash
   git lfs install
   git lfs track "data/cpv_embeddings.npz"
   git add .gitattributes data/cpv_embeddings.npz
   git commit -m "Track embeddings via LFS"
   git push
   ```
   Streamlit Cloud supports Git LFS on the free tier.
2. On [share.streamlit.io](https://share.streamlit.io) → **New app** → point at this repo, branch, and `app.py`.
3. Streamlit Cloud will install `requirements.txt` and start the app. Cold start downloads the model once (~1–2 min); after that it's cached.

Resource note: the embedding model needs ~1 GB RAM to load. The free tier provides 1 GB, which is workable but tight — if you hit OOM, upgrade to Streamlit Cloud's paid resource boost or move to Hugging Face Spaces (Free tier provides 16 GB).

## Tuning

- **Confidence threshold** (sidebar slider, default `0.45`): lower it to get more recall, raise it for higher precision. Calibrate it on a handful of known tenders for your domain.
- **Language aggregation**: `max` (default) is more responsive to mixed-language tenders; `mean` is smoother but tends to dilute signal.

## Future extensions

- **Phase 2 (file uploads)** — already wired up: the app accepts `.txt`, `.pdf`, `.docx`, `.xlsx`. Text is extracted locally with `pypdf`, `python-docx`, and `openpyxl`. Nothing leaves the server.
- **Phase 3 (LLM-assisted explanations)** — hook for sending top-N matches to Claude for natural-language justifications. The classifier itself remains deterministic; Claude only ever explains, never selects.

## Why "no hallucination"?

A language model could produce a plausible-looking CPV code that doesn't actually exist (e.g. `45219000` when only `45210000` and `45220000` are real). This app never generates codes — it only ranks the closed set of 1,321 real four-digit divisions from the CSV, and the final result is filtered against that same CSV before display. The worst possible failure mode is "wrong but real code," never "fabricated code."
